# 교훈창고

기술 이름(slug): `gyohun-changgo`

## 한 줄 소개
사업 평가에서 얻은 작동요인·비작동요인·개선사항을 분야별로 분류해 축적하고, 이행까지 추적하는 평가 교훈 관리 시스템 — 같은 팀 동료들이 함께 쓴다.

## 핵심 흐름
1. 사용자가 평가에서 발견한 교훈을 분야·구분(작동요인/비작동요인/개선사항)과 함께 입력한다
2. 성과관리 담당자가 환류과제 구분·이행주체·이행 모니터링 결과를 추가한다
3. 축적된 교훈을 분야·구분·사업명 등으로 필터·검색해 새 사업에 참고한다

## 교훈 분류 체계 (4 Level)
- **Level 1 — 분야**: 교육 (향후 보건, IT 등 확장 가능)
- **Level 2 — 세부분야**: 유아교육 / 초등교육 / 중등교육 / 고등교육 / 직업훈련 / 인재양성 / 디지털교육
- **Level 3 — 성과영역**: 교육접근성 / 학습성과 / 정책·제도 개선
- **Level 4 — 사업요소**: 교육시설 / 교사 연수 / 교육 기자재 / 교사학습공동체 / 학교운영위원회

입력 시 Level 1 선택 → Level 2 드롭다운 갱신 → Level 3 → Level 4 순서로 단계적 선택.
필터·검색도 이 4단계를 기준으로 한다.

## 데이터 (Supabase 테이블)
- **lessons**
  - `id` uuid — 기본 키
  - `level1` text — 분야 (예: 교육)
  - `level2` text — 세부분야 (예: 초등교육, 디지털교육)
  - `level3` text — 성과영역 (예: 교육접근성, 학습성과)
  - `level4` text — 사업요소 (예: 교사 연수, 교육시설)
  - `project_name` text — 사업명
  - `evaluation_name` text — 평가명 (예: 중간평가, 종료평가)
  - `factor_type` text — 구분 ('작동요인' / '비작동요인' / '개선사항')
  - `content` text — 교훈 및 시사점 (긴 텍스트)
  - `author` text — 작성자/작성일
  - `follow_up_type` text — 환류과제 구분 (nullable)
  - `responsible` text — 이행주체 (nullable)
  - `monitoring_result` text — 이행 모니터링 결과 (nullable)
  - `monitoring_status` text — 이행 상태 ('반영' / '미반영' / '진행중', nullable)
  - `created_at` timestamptz — 생성일
  - 관계: 없음
